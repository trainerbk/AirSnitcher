                            AirSnitch Bypass Tests
                            
                                modified wpa_supplicant frame injection & GTK manipulation
                            
                        

                        

                            These tests use airsnitch.py with the modified wpa_supplicant to probe AP isolation
                            at the 802.11 frame level. Each test injects real attack frames — if the AP is
                            vulnerable, other clients on the network ARE affected.
                        


                        
                        

                            

                                

                                    

                                        Group Temporal Key (GTK)
                                    

                                    

                                        Shared by all clients on this BSS — enables broadcast frame injection
                                    

                                

                                
                            

                            

                        


                        
                        

                            

                                
GTK Shared Key

                                
Verifies all clients share the same Group Temporal Key — precondition for GTK abuse attacks

                                
                            

                            

                                
ARP Poisoning (C2C)

                                
Sends spoofed ARP via frame injection to redirect victim traffic through the attacker

                                
                            

                            

                                
IP-Layer Bypass

                                
Bounces packets through the gateway to reach isolated clients at L3 — gateway is not subject to isolation

                                
                            

                            

                                
Port Steal (Downlink)

                                
MAC spoofing to hijack the AP's forwarding table and intercept victim's downlink traffic

                                
                            

                            

                                
Port Steal (Uplink)

                                
MAC spoofing to intercept victim's outbound traffic before it reaches the gateway

                                
                            

                            

                                
Broadcast Reflection

                                
Tests if broadcast frames reach isolated clients — if vulnerable, enables mass ARP poisoning of all clients

                                
                            

                            

                                
Ethernet Layer

                                
Direct L2 Ethernet frame delivery between isolated clients via crafted 802.11 frames

                                
                            

                            

                                
GTK Injection

                                
Injects GTK-encrypted broadcast frames — bypasses AP forwarding logic entirely (radio-to-radio)

                                
                            

                            

                                
Multicast (C2M)

                                
Tests if multicast group traffic crosses isolation boundaries to reach other clients

                                
                            

                            

                                
Multicast IP (C2M-IP)

                                
Tests IP-layer multicast delivery to isolated clients via crafted multicast frames

                                
                            

                        

                        

                            
                        


                        
                        

                            

                                Active Exploitation
                            

                            

                                Run “Broadcast Reflection” first to verify this attack vector is viable.
                                The attack enables IP forwarding and iptables MASQUERADE automatically.
                            

                            

                                

                                    
1. Launch MITM

                                    
Enables IP forwarding, iptables NAT, then sends broadcast ARP poison claiming to be the gateway

                                    
                                

                                

                                    
2. Verify Capture

                                    
Runs 10s tcpdump capture to check if traffic from other clients is being intercepted

                                    
                                

                                

                                    
3. Stop & Cleanup

                                    
Disables forwarding, removes iptables rule, sends corrective ARP to restore real gateway
